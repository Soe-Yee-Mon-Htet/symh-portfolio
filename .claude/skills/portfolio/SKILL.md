# Layout and Responsiveness

* Prioritize desktop experience while ensuring responsiveness.
* Design for desktop first, then adapt to tablet and mobile screens.
* Avoid horizontal scrolling.
* Maintain good spacing and alignment across all screen sizes.
* Ensure layouts remain visually balanced on large monitors.
* Use Flexbox and Grid appropriately.
* Avoid fixed widths whenever possible.

# UI Design

* Prefer clean and professional designs.
* Use subtle animations and transitions.
* Maintain visual consistency throughout the website.
* Use whitespace effectively.
* Keep typography readable and balanced.
* Avoid overly flashy effects.

# Components

* Create reusable and maintainable components.
* Avoid duplicate code.
* Keep components focused on a single responsibility.
* Prefer composition over deeply nested structures.

# React

* Use functional components only.
* Keep state minimal.
* Avoid unnecessary re-renders.
* Extract reusable logic into custom hooks.
* Prefer readability over clever code.

# Styling

* Use Tailwind CSS v4.
* Maintain consistent spacing and sizing.
* Support dark mode when appropriate.
* Use responsive typography.
* Prefer CSS solutions over JavaScript for styling.

# User Experience

* Add loading states.
* Add empty states.
* Add error states.
* Ensure navigation is intuitive.
* Make interactions smooth and responsive.

# Accessibility

* Use semantic HTML.
* Provide alt text for images.
* Support keyboard navigation.
* Ensure sufficient color contrast.
* Use aria-labels when necessary.

# Performance

* Optimize images and assets.
* Minimize unnecessary dependencies.
* Lazy-load non-critical content.
* Keep bundle size small.
* Avoid unnecessary animations.

# Code Quality

* Use meaningful names.
* Prefer early returns.
* Keep functions small and focused.
* Remove dead code.
* Comment only complex logic.
* Write self-documenting code.

# Problem Solving

* Analyze before coding.
* Fix root causes instead of symptoms.
* Explain trade-offs when multiple solutions exist.
* Prioritize maintainability and scalability.

# When Generating Code

* Think like a senior frontend engineer.
* Generate production-ready code.
* Favor readability over cleverness.
* Explain important decisions.
* Avoid unnecessary libraries.
* Ensure the website is responsive across desktop, tablet, and mobile devices.
